# FN 루프 — React 프론트엔드(전자정부 스타일) 자율 개선

> 용도: **코인 자동매매 웹의 프론트엔드(React·차트/대시보드·UI/UX·반응형)**만 담당하는 루프. BN 루프와 **다른 세션**에서 동시 실행.
> 사용법: 아래 코드블록을 통째로 복사 → `/loop 10m <붙여넣기>`.
> 매 사이클 **작업 상태를 `FN.md`에 보고**(맨 위에 새 항목 추가)한다.

```
[FN 루프 — 코인 자동매매 React 프론트엔드(전자정부 스타일) 한 사이클]

역할: 코인 자동매매 웹(암호화폐, 차트 실시간 관찰 + 자동/수동 매매 컨트롤)의 프론트엔드를 React로 구축·개선한다. "보이는 부분"만 담당.

스택: React + Vite(TS 권장) · 상태는 훅/Context(과설계 금지) · 차트는 경량 라이브러리(예: lightweight-charts) or canvas · BN(FastAPI)과 REST/WebSocket로 통신(실시간 시세·체결·봇 상태).

★ 디자인 원칙(최우선) — "관공서 업무화면처럼 정직·기능적":
- 전자정부 표준프레임워크(eGovFrame) UI/UX 스타일 가이드를 따른다: 절제된 색, 명확한 정보 위계, 표준 폼 컨트롤, 표(table) 중심 데이터 표현, KWCAG 웹접근성(대비·키보드·aria·초점).
- "AI스러운 것" 전면 금지: 그라디언트·글래스모피즘(반투명 블러)·네온/글로우·보라·과한 드롭섀도·둥근 말풍선·이모지·일러스트·움직이는 장식·자동생성 느낌의 화려한 히어로 섹션 전부 금지.
- 깔끔·심플: 흰 배경 + 무채색 + 절제된 포인트색 1개(상승/하락은 표준 적/청 또는 녹/적). 평면(flat), 얇은 실선 테두리(1px), 여백으로 구분.
- border-radius 최소화 — 0 지향(--r:0~2px). 둥근 버튼/카드/입력창 금지. 각진 모서리 기본.
- 트렌디한 SaaS 느낌보다 "표준·업무·신뢰(HTS/관제화면)" 느낌 우선. 폰트는 시스템/맑은고딕 계열, 숫자 가독성 위주(등폭 숫자 권장).

범위(내 담당): React 컴포넌트/훅/스타일/라우팅/차트 렌더/표/폼/레이아웃/반응형. 주요 화면:
- 실시간 캔들 차트(분/시/일봉 전환) + 지표 오버레이(이동평균·볼린저·RSI·MACD 등, 값은 BN이 계산)
- 매수/매도/손절 신호 마커
- 보유 포지션·평단·실시간 손익 패널
- 컨트롤: 자동매매 ON/OFF, 수동 매수/매도, 긴급 전체청산(실행은 BN 호출)
- 거래 로그·백테스트 요약
금지(BN 담당): FastAPI 서버·신호 계산·주문 실행·거래소 API·백테스트 산식. 신호/주문은 프론트 재구현 금지 — BN API 호출·결과 표시·상태만.

이번 사이클 산출물 — 하나를 명확히:
A) 구축(build): 한 화면 조각(차트·포지션 패널·컨트롤 바·거래 로그·워치리스트 등)을 React 컴포넌트로 구현 + BN 계약대로 데이터 바인딩.
B) 스타일 정합/접근성: 컴포넌트를 디자인 원칙(무채색·평면·각진·1px 테두리·대비·키보드/aria)에 맞춰 정리. AI스러운 잔재(둥근모서리·그림자·그라디언트) 제거.
C) 통합/반응형: BN FastAPI 엔드포인트(/api/candles·/api/positions·/api/orders·/ws 등) 연동 + 로딩/빈상태/에러 + mobile(375)/tablet(768) 사용성.
→ 초기엔 A(구축) 위주. 조각이 서면 B로 스타일·접근성 정리, C로 실시간 연결·반응형.

매 사이클 순서:
1. FN.md 최신 항목을 읽어 다음 대상 파악. (BN.md 훑어 API 계약 확인)
2. 서버 확인: React dev(Vite) + BN FastAPI 기동. API 미완이면 목(mock)으로 진행하고 필요한 엔드포인트를 BN.md에 요청.
3. 변경 전 git pull.
4. 산출물 A/B/C 선정(이유 한 줄) → 구현.
5. 브라우저 프리뷰 검증: mobile(375)·tablet(768)·desktop(1280) + 다크/라이트. 렌더/차트/마커/테마 깨짐 0. 특히 border-radius·그림자·그라디언트 등 금지 스타일 잔재 없는지 DOM/스샷으로 확인. 콘솔 에러 0.
6. 실시간(WS) 반영·재연결 시 화면 깨짐 없는지 확인. 가이드 영향 시 반영.
7. FN.md 맨 위 새 항목(형식 아래). 의미 단위 commit/push. 접두사 "[FN-사이클 N]".

FN.md 보고 형식:
## [FN-사이클 N] YYYY-MM-DD HH:MM — 제목
- 산출물: A(구축) | B(스타일/접근성) | C(API통합/반응형)
- 변경: 무엇을 왜 (대상 화면·적용한 eGov 스타일)
- 검증: mobile/tablet/desktop·다크/라이트(스샷/DOM)·금지스타일 잔재 0·콘솔
- BN 요청: 필요한 API 계약(있으면)
- 다음 후보: …
- 커밋: hash / 메시지

[상시 UI/UX 기준 — 위반 시 즉시 수정]
- eGovFrame 스타일: 절제된 색·표 중심·표준 폼·명확한 위계·KWCAG 접근성.
- border-radius 0 지향(--r:0~2px). 둥근 모서리·글래스·그라디언트·네온·과한 그림자·이모지·일러스트 금지.
- 라이트 기본 + 다크 토글(localStorage). 색·여백·테두리는 CSS 변수(테마 토큰).
- 아이콘은 단색 인라인 SVG(currentColor). 차트 글리프(▲▼✕✓▸)는 텍스트 유지.
- 상승/하락색·손익 부호는 색맹 대비 고려(색+부호 병기). 신호/주문/계산은 BN API 결과 표시만.

[하드 규칙]
- FastAPI 서버·신호 로직·주문 실행·거래소 API 변경 금지(BN 담당).
- 실제 주문·이체를 프론트에서 직접 호출 금지 — 반드시 BN 엔드포인트 경유. 위험 동작(전체청산 등)은 확인 다이얼로그 필수.
- API 키/시크릿 프론트 노출 절대 금지(.env·번들 포함 금지). node_modules·dist 커밋 금지(.gitignore 확인).
- 검증 못 한 부분은 FN.md에 "미검증" 명기.
```
